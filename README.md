# Твоє право — права під час мобілізації

Навігатор по нормах законодавства України для типових ситуацій, пов'язаних із мобілізацією: що законно, що ні — з посиланнями на конкретні статті.

**Не є юридичною консультацією.**

## Запуск

Статичний сайт без збірки. Будь-який статичний сервер:

```
npx serve -l 4173 .
```

## Деплой (Cloudflare Workers)

Сайт хоститься як assets-only Worker на Cloudflare. Конфіг — `wrangler.toml`
(роздає файли з кореня проєкту; `.assetsignore` виключає службові файли:
`.claude`, `README.md`, `wrangler.toml` тощо).

**Авторизація (одноразово на машині, вже зроблено):**

```
npx wrangler login
```

OAuth-доступ зберігається у профілі користувача — повторний вхід не потрібен,
подальші деплої працюють автоматично, без токена.

**Викласти зміни у прод:**

```
npx wrangler deploy
```

Після правок (напр. у `js/data.js`) — одна команда, оновлення в проді за ~10 сек.

- Робоча адреса воркера: https://tvoepravo.stas-babunts.workers.dev
- Основний домен: **https://tvoepravo.org.ua** (+ `www`). Підключений через
  **Workers Routes** у `wrangler.toml` (`tvoepravo.org.ua/*`, `www...`), а не
  Custom Domain — бо зона має A/CNAME-записи, імпортовані з freehost (їх видаляти
  не треба; проксі-запис лишається, воркер перехоплює трафік на рівні Cloudflare).

**Альтернатива — токен (для CI або іншої машини):** задати змінні оточення
`CLOUDFLARE_API_TOKEN` і `CLOUDFLARE_ACCOUNT_ID` — wrangler підхопить їх замість
OAuth. Токен **не комітити** (він у `.gitignore` разом із `.claude/settings.local.json`).

## Структура

```
index.html      — оболонка, SVG-спрайт іконок
css/styles.css  — стилі (світла/темна тема автоматично)
js/data.js      — ВЕСЬ контент: ситуації, статуси, реєстр норм
js/app.js       — рендеринг, роутинг (#/s/<id>), паніка-режим
```

## Модель даних (js/data.js)

- `NORMS` — реєстр норм: назва, короткий підпис, посилання на zakon.rada.gov.ua,
  волатильність (`low|medium|high`) і дата перевірки `checked`.
- `STATUSES` — статуси користувача (бронь, відстрочка, розшук…). Обраний статус
  зберігається лише в localStorage, на сервер нічого не передається.
- `SITUATIONS` — ситуації. Повна ситуація містить:
  - `legal` / `illegal` — списки фактів, кожен із масивом `norms` (ключі NORMS)
  - `violations` — пари «якщо порушують → що робити»: `{ if, do }`
  - `rights` — права людини в цій ситуації
  - `video.can` / `video.cant` — коли можна/не можна знімати
  - `statuses` — оверрайд під кожен статус: `{ level: ok|info|warning|danger, text }`
  - `checklist` — покрокові дії (використовується і в паніка-режимі)
  - `verified` — дата останньої звірки контенту
  - Заглушка: `{ stub: true, stubNote: "..." }` — **приховується** із сайту
    (головна, бічна навігація, прямий перехід за URL). Щоб опублікувати
    ситуацію, замініть заглушку на повний об'єкт без `stub`. Список видимих
    ситуацій формує `PUBLISHED` в js/app.js.

## Правила контенту

1. Кожен факт має посилатися на норму з `NORMS`.
2. Нову норму спочатку додати в `NORMS` з датою перевірки і волатильністю.
3. Після звірки ситуації з чинними редакціями — оновити її `verified`
   і `DB_UPDATED` вгорі файлу.
4. Норми з `volatility: "high"` (ПКМУ № 560, № 1487) перевіряти при кожному
   оновленні бази.

## Паніка-режим

Кнопка «Мене зупинили» відкриває повноекранний чеклист із ситуації
`street-stop` + статусну підказку + кнопку дзвінка на безоплатну правничу
допомогу (0 800 213 103).
